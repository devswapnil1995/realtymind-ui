## ğŸ’¡ Developer Notes

### For Developers Using This Codebase

1. **Always use design system variables**
   - Don't hardcode colors, use `$primary-color`, etc.
   - Use spacing variables for consistency

2. **Follow the design system**
   - Use existing mixins for common patterns
   - Keep animations performant (transform & opacity)
   - Test responsiveness at all breakpoints

3. **Maintain accessibility**
   - Use semantic HTML
   - Add ARIA labels where needed
   - Test with keyboard navigation
   - Verify color contrast

4. **CSS Best Practices**
   - Use BEM naming convention
   - Keep selectors simple
   - Use SCSS nesting appropriately
   - Comment complex styling

5. **Testing Checklist**
   - Desktop (1920px)
   - Tablet (768px)
   - Mobile (375px)
   - Dark mode (if added)
   - Screen reader
   - Keyboard navigation
